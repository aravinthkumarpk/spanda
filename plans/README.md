## Findings considered and rejected (so they aren't re-audited)
- **File-length violations (13 files >500 lines)** — REJECTED. `wc -l` counts raw lines; ESLint `max-lines` skips blanks/comments and **lint is green**, so there is no actual violation (e.g. `workflows.ts` = 642 raw / ~500 code). Not a finding.
- **"toolInFlight keeps a session 'running' forever"** — REJECTED/downgraded. `classify.ts:37` returns `done` at `doneMs` (6h) *before* the `toolInFlight` check at `:40`, so an abandoned run becomes `done`, not stuck. By-design window.
- **Command injection via comment text / bd exec** — REJECTED. `bd-internal.ts` uses `execFile` with an argv array (no shell); comment text can't escape argument boundaries. Safe.
- **Path traversal in the artifact viewer** — REJECTED. `artifact-loader.isSafeRelPath` validates each segment, rejects `.`/`..`/absolute/backslash/non-`.html` before any fs touch (e2e-verified: `..%2F..%2Fetc%2Fpasswd` leaks 0 lines).
- **XSS via inlined artifact/daily HTML** — REJECTED (low). Those files are system-authored (the synth + html-artifacts skill on the same box), not user input; comment text renders via JSX auto-escape, never as raw HTML.
- **Missing per-route auth** — NOTED, not planned. Acceptable given the single-operator Cloudflare-Access deployment; would become critical only if the trust boundary changes.
- **npm audit highs (vitest beta XSS, transitive esbuild/hono)** — NOTED, not planned. Dev/test-time tooling, not the production path; revisit on a dependency-bump pass.
