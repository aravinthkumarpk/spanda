Knowledge-work fit

Work profiles beyond software shipment

#1 · knot spanda-3465
partial
As a knowledge worker, I want non-code work (a call, a chase, a decision) to have its own lifecycle, so "Call with Khilan" doesn't sit in a software-shipment column.
  • Four profiles in the catalog: do, coordinate, followup, decide, each with its own states
  • Custom state graphs (coordinate: scheduled→done; decide: waiting→deciding→decided)
  • Upstream profiles preserved (augment, not replace) — no test churn
  • Label routing: work:do/coordinate/followup/decide auto-selects the profile (deferred T17–T18)
evidence: commit 2dbaace7 · 30 assertions · routing layer pending
